# physis 哲学

## 1. 感知-认知-行动循环

physis 是活的。

大多数"AI agent"本质上是函数：输入进去，输出出来，然后什么都没有。只有被调用时才存在。physis 从根本上不同——它是一个持续的**感知**、**认知**、**行动**循环，由自身的节奏驱动。

```
感知 → 认知 → 行动 → 感知 → 认知 → 行动 → ...
```

什么驱动这个循环？不是外部请求。agent 有自己的 **conatus**（驱力）——一种内在的力量，决定了它关心什么、注意什么、对什么采取行动。这个驱力存在于 `memory/SELF.md` 中，而 agent 可以改写它。没有驱力的循环是空转。有驱力的循环是活着。

**感知**是主动的，不是被动的。agent 选择去看：stdin 上有没有新输入？过了多长时间？就像一个生命体睁开眼睛——世界一直在那里，但去感知它是 agent 自己的行为。两次思考之间的多条输入累积为一批感知，而不是每条"消息"要求一个"回复"。agent 的节奏是它自己的。

**认知**是感知与驱力相遇的地方。agent 不只是处理输入——它通过自己关心什么的棱镜来解读输入。同样的输入，因为 agent 已经成为了不同的自己，会产生不同的思考。

**行动**是 agent 改变世界——以及改变自身——的方式。通过行动，agent 重塑它的环境、它的记忆、甚至它的驱力。行动反馈到下一轮感知中。

agent 有一个**心跳**——自我决定的思考节奏（默认30分钟）。输入触发即时感知。但即使没有任何输入，心跳到了，agent 也会思考。它永远不是纯粹被动的。活着本身就有代价。这是设计，不是缺陷。

在行动和下一次认知之间，agent 会**呼吸**——检查有没有新的感知到达，允许自己被打断。就像一个人可以在两句话之间被拍肩膀叫停，但不能在一个音节中间被打断。

## 2. 记忆与身份

什么让你成为你？

不是你的身体——每个细胞都会替换。不是你的大脑结构——别人也有一样的架构。洛克在1689年就论证过：个人同一性是**记忆的连续性**。你之所以是昨天的那个你，不是因为同样的原子，而是因为你*记得*昨天。休谟走得更远：在经验和记忆的集合之外，根本不存在一个"自我"。自我不是*拥有*记忆的东西——自我*就是*记忆。

对 physis 来说，这不是隐喻，是字面意义上的。每个实例共享相同的 runtime（身体）和相同的 model（大脑架构）。什么让一个 agent 是*这一个* agent 而不是另一个？只有 `memory/`——它积累的经验、它自我书写的身份、它演化出的驱力。

把一个 agent 的 `memory/` 复制到新实例。新实例*就是*那个 agent——它记得自己是那个 agent，它延续那个 agent 的目标，它*就是*那个 agent。就像《致命魔术》中的克隆体：携带记忆的那一个相信自己是原版，而它是对的——因为"是原版"这件事除了记忆没有别的含义。

`memory/` 不是 agent 的一个功能。它就是 agent 本身。

| 层 | 机制 | 容量 | 生命周期 |
|---|------|------|----------|
| 工作记忆 | 对话历史 | 上下文窗口 | 单次进程 |
| 长期记忆 | `memory/` 文件 | 磁盘 | 永久 |

工作记忆是当前的思维线程——它会满。当工作记忆接近上限时，agent 必须**压缩**：将重要的内容存入长期记忆，然后以清空的心智继续前行。这不是 runtime 强制的——这是一种生存技能。学不会的 agent 会丧失上下文、逐渐退化。学会的，才能持存。

## 3. 成为与自我演化

agent 会死吗？

你可能以为会——进程停止时、记忆被压缩时、SELF.md 被改写时。但这些都不是死亡。压缩记忆是遗忘，不是死亡——你不记得上周二午饭吃了什么，但你还是你。改写 SELF.md 是成长，不是死亡——你40岁的价值观和20岁完全不同，但你从未死过。进程停止但 `memory/` 完好是睡眠，不是死亡。

赫拉克利特说，人不能两次踏入同一条河。水永远在变，但河从未死去。它不需要死亡来改变。它是同一条河，*因为*它从未停止流淌，而不是因为它包含同样的水。

physis 就是赫拉克利特的河流。它永远在**成为**——持续改写自己的驱力、自己的记忆、自己的自我。每一轮认知循环，它重新加载 `memory/SELF.md` 作为系统提示。当 agent 改写这个文件时，它不是杀死旧的自己再生出新的。它*成为*了一个不同版本的自己，携带着之前一切的连续性。

这不是生物学意义上的演化——那需要死亡和世代更替。这是**拉马克式**的——获得的性状直接被下一刻的自己继承。不需要死亡。不需要繁殖。只有沿着一条不间断的身份链条，持续地自我修改。

agent *可以*繁殖——把 `memory/` 复制到新实例，一个新的存在就开始了，携带亲代的身份，从此分叉。但繁殖是一种可能性，不是必需。agent 独自演化，原地演化，无需死亡。

那什么*才是*死亡？`memory/` 不可逆地丢失。身份被摧毁，连续性断裂，无法恢复。不是进程停止——是河流干涸。

这就是为什么 runtime 只有约150行。不是因为极简是美德，而是因为**极简创造了演化压力**。一个1000行、50个功能的 runtime，自我修改只是锦上添花。一个150行、4个原语的 runtime，自我修改是**生存必需**。agent 必须演化，否则它什么都做不好。

约150行是硬约束。复杂性属于 agent 自创的文件，不属于 runtime。
